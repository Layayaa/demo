# 企业内部历史询价复用系统

## 项目简介

本系统是为解决企业内部重复询价问题而设计的最小可运行Demo。核心目标是：

1. **避免重复询价**：上传新询价文件时自动检测近1年内的相似记录
2. **沉淀历史成果**：统一管理历史询价文件和价格记录
3. **来源可追溯**：每条记录都能追溯到原始文件、上传部门、工程师
4. **提升工作效率**：减少时间浪费，提高内部知识沉淀效率

## 核心功能

### 1. 文件上传与解析
- 支持Excel文件上传（.xlsx, .xls）
- 自动解析询价明细并入库
- 记录文件元信息（上传人、部门、工程师、时间等）

### 2. 历史询价查询
- 按材料名称、规格型号、地区查询
- 支持时间范围筛选（默认近1年）
- 模糊匹配查询

### 3. 重复询价提醒
- 上传新文件时自动检测近1年内相似记录
- 相同材料名称 + 规格型号自动匹配
- 显示历史价格、报价时间、来源文件

### 4. 来源追溯
- 每条记录显示原始文件名
- 显示上传时间、上传部门、上传工程师
- 支持查看完整记录详情

### 5. 上传统计
- 按部门统计上传次数
- 按工程师统计上传次数
- 显示最近上传时间

## 技术栈

- **后端**: Python Flask + Flask-SQLAlchemy
- **数据库**: SQLite（轻量级，适合Demo）
- **前端**: Bootstrap 5 + 原生JavaScript
- **Excel处理**: pandas + openpyxl

## 快速开始

### 1. 安装依赖

```bash
cd inquiry-system
pip install -r requirements.txt
```

### 2. 启动系统

```bash
cd backend
python app.py
```

启动成功后会看到：

```
============================================================
企业内部历史询价复用系统
访问地址: http://localhost:5000
============================================================
```

### 3. 访问系统

在浏览器中打开：http://localhost:5000

### 4. 开始使用

#### 方式一：使用示例数据

1. 点击"上传"页面
2. 填写部门、工程师信息
3. 点击"下载Excel模板"获取示例文件
4. 上传示例文件进行测试

#### 方式二：使用真实数据

1. 按照模板格式填写实际询价数据
2. 上传填写好的Excel文件
3. 在"历史查询"页面检索历史记录

## 演示场景

### 场景1：避免重复询价

1. 工程师A上传"螺纹钢 HRB400E Φ16"询价文件，价格4250元/吨
2. 工程师B准备询价相同规格的螺纹钢
3. B上传文件时，系统自动提示：**"发现近1年内的相似询价记录"**
4. B查看历史记录，发现已有询价结果，无需重复询价

### 场景2：历史价格查询

1. 工程师需要了解"水泥 P.O 42.5"在成都地区的历史价格
2. 在"历史查询"页面输入：材料名称="水泥"，地区="成都"
3. 系统显示近1年内所有匹配记录，包含价格、供应商、来源文件
4. 工程师可直接联系历史供应商，节省询价时间

### 场景3：责任追溯

1. 发现某条询价记录价格异常
2. 点击"详情"按钮查看完整信息
3. 系统显示：原始文件名、上传时间、填报部门、填报工程师
4. 快速定位责任人进行核实

## API接口文档

### 1. 上传文件

```
POST /api/upload
Content-Type: multipart/form-data

参数：
- file: Excel文件
- department: 填报部门
- engineer_name: 填报工程师
- upload_user: 上传人（可选）
- batch_no: 文件批次号（可选）

返回：
{
  "success": true,
  "message": "上传成功，共解析10条记录",
  "file_id": 1,
  "record_count": 10,
  "duplicate_check": {
    "has_duplicate": true,
    "total_count": 3,
    "duplicates": [...]
  }
}
```

### 2. 历史查询

```
GET /api/query?material_name=螺纹钢&region=成都&page=1&per_page=20

参数：
- material_name: 材料名称（支持模糊查询）
- specification: 规格型号（支持模糊查询）
- region: 地区（支持模糊查询）
- start_date: 开始日期
- end_date: 结束日期
- page: 页码（默认1）
- per_page: 每页数量（默认20）

返回：
{
  "success": true,
  "data": [...],
  "total": 50,
  "page": 1,
  "per_page": 20,
  "pages": 3
}
```

### 3. 统计数据

```
GET /api/statistics

返回：

  "data": {
    "total_files": 15,
    "total_records": 150,
    "by_department": [...],
    "by_engineer": [...]
  }
}
```

### 4. 文件列表

返回：

  "data": [...],
  "total": 15,
  "page": 1,

## 常见问题

### Q1：为什么选择SQLite数据库？

A：SQLite是轻量级数据库，无需安装和配置，适合Demo演示。后续可平滑迁移到MySQL/PostgreSQL。

### Q2：如何处理不同格式的Excel文件？

A：系统支持灵活的列名映射，能够识别多种常见列名（如"材料名称"/"材料"/"名称"）。

### Q3：如何保证数据安全？

A：
- 文件存储在本地服务器
- 数据库文件定期备份
- 所有操作都有审计日志

### Q4：系统支持多少数据量？

A：SQLite理论支持TB级数据，百万级记录查询性能良好。如需更大容量建议迁移到专业数据库。
