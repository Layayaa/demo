# 企业内部历史询价复用系统

## 项目简介

本系统实现企业内部历史询价数据的复用，遵循"先查后询 → 询完必录 → 录后可溯 → 溯后可用"的业务闭环，帮助企业避免重复询价、提高采购效率、实现价格数据的价值沉淀。

## 技术架构

```
前端：Bootstrap 5 + 原生 JavaScript
后端：Flask + SQLAlchemy
数据库：SQLite（三库分离）
├── inquiry_file.db  - 文件管理
├── price_record.db  - 询价明细
└── upload_audit.db  - 审计留痕
```

## 快速启动

### 1. 环境要求

- Python 3.8+
- pip

### 2. 安装依赖

```bash
cd backend
pip install flask flask-sqlalchemy flask-cors pandas openpyxl
```

### 3. 启动服务

```bash
cd backend
python app.py
```

启动成功后访问：http://localhost:5000

### 4. 数据库初始化

首次启动会自动创建三个数据库文件：
- `database/inquiry_file.db` - 管理每次上传的原始文件
- `database/price_record.db` - 管理每条询价明细记录
- `database/upload_audit.db` - 形成上传与责任留痕

## 功能演示

### 一、避免重复询价

**操作步骤：**

1. 进入系统首页，在"自然语言查询"框输入查询内容
   - 示例：`查询螺纹钢价格`、`德阳商品混凝土C30多少钱`

2. 系统自动解析查询意图，返回近1年内的历史询价记录

3. 查看查询结果，确认是否已有相同材料的历史报价

4. 如果存在历史报价，可直接使用，避免重新询价

**演示场景：**
```
用户输入：查询螺纹钢价格
系统返回：
  - 螺纹钢 HRB400E Φ16 | 4250元/吨 | 采购部 张三 | 2026-03-15
  - 螺纹钢 HRB400E Φ20 | 4300元/吨 | 采购部 张三 | 2026-03-15
```

### 二、查询近1年历史文件

**操作步骤：**

1. 使用"传统查询"表单进行精确查询

2. 可按以下条件筛选：
   - 材料名称（支持模糊查询）
   - 规格型号（支持模糊查询）
   - 地区
   - 时间范围（默认近1年）

3. 点击"传统查询"按钮获取结果

4. 点击记录详情可查看完整信息和来源文件

**演示场景：**
```
查询条件：材料名称=混凝土，地区=成都，时间=近1年
系统返回：50条符合条件的询价记录
```

### 三、追溯上传责任主体

**操作步骤：**

1. 在查询结果中点击任意记录的"详情"按钮

2. 详情弹窗显示完整信息，包括：
   - 材料名称、规格型号、单价
   - 报价时间、地区、供应商
   - **来源追溯信息**：
     - 原始文件名
     - 上传时间
     - 填报部门
     - 填报工程师

3. 每条记录都有明确的责任主体，便于后续核实

**演示场景：**
```
记录详情：
  材料：商品混凝土 C30
  单价：485元/方
  来源文件：2026年3月询价汇总表.xlsx
  上传时间：2026-03-20 14:30:00
  填报部门：采购部
  填报工程师：张三
```

### 四、上传新询价文件

**操作步骤：**

1. 在页面右侧"上传新询价"区域填写信息
   - 填报部门（必填）
   - 填报工程师（必填）
   - 报价有效期（默认1年）

2. 点击或拖拽上传 Excel 文件
   - 支持 .xlsx, .xls 格式
   - 最大 16MB

3. 系统自动解析文件并入库

4. 上传成功后显示解析结果统计

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 上传Excel文件 |

## 数据库设计

### inquiry_file.db（文件数据库）

| 字段 | 类型 | 说明 |
|------|------|------|
| file_id | Integer | 主键 |
| file_name | String | 原始文件名 |
| upload_time | DateTime | 上传时间 |
| upload_user | String | 上传人 |
| department | String | 填报部门 |
| engineer_name | String | 填报工程师 |
| record_count | Integer | 解析记录数 |

### price_record.db（明细数据库）

| 字段 | 类型 | 说明 |
|------|------|------|
| record_id | Integer | 主键 |
| file_id | Integer | 关联文件ID |
| material_name | String | 材料名称 |
| specification | String | 规格型号 |
| price | Float | 单价 |
| region | String | 地区 |
| quote_date | Date | 报价时间 |
| reference_count | Integer | 引用次数 |

### upload_audit.db（审计数据库）

| 字段 | 类型 | 说明 |
|------|------|------|
| audit_id | Integer | 主键 |
| file_id | Integer | 关联文件ID |
| upload_time | DateTime | 上传时间 |
| upload_user | String | 上传人 |
| department | String | 填报部门 |
| engineer_name | String | 填报工程师 |
| status | String | 状态 |

## 注意事项

1. **数据有效期**：上传时可设置报价有效期，过期记录会标记提醒
2. **引用计数**：每次查询引用记录时，系统自动增加引用计数
3. **数据安全**：所有上传操作都有审计留痕，可追溯责任
