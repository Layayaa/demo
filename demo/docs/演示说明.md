# 企业内部历史询价复用系统 - 演示说明文档

## 一、演示准备

### 1. 环境检查

确保以下环境已准备就绪：

- Python 3.8+ 已安装
- 依赖包已安装（`pip install -r requirements.txt`）
- 端口5000未被占用

### 2. 启动系统

```bash
cd inquiry-system/backend
python app.py
```

看到以下提示表示启动成功：

```
============================================================
企业内部历史询价复用系统
访问地址: http://localhost:5000
============================================================
```

### 3. 准备演示数据

系统已内置样例数据文件，位于 `sample_data/询价模板_示例数据.xlsx`

包含10条示例询价记录，涵盖：
- 螺纹钢、水泥、砂石、混凝土等常见材料
- 成都、绵阳、德阳、宜宾、泸州等四川地区
- 不同规格型号和价格

## 三、关键演示要点

### 1. 如何避免重复询价？

**核心机制**：
- 上传新文件时，系统自动检测近1年内相似记录
- 匹配规则：材料名称 + 规格型号完全相同
- 立即提醒用户，显示历史价格和来源

**价值**：
- 减少重复劳动
- 节省询价时间
- 提高工作效率

### 2. 如何查询近1年历史文件？

**查询方式**：
- 默认查询近1年记录（365天）
- 支持6个月、3个月、全部、自定义时间范围
- 支持材料名称、规格型号、地区的模糊查询

**价值**：
- 快速找到历史询价结果
- 复用已有的询价成果
- 减少从头询价的工作量

### 3. 如何追溯到上传责任主体？

**追溯路径**：
- 每条记录关联原始文件
- 每个文件记录上传人、部门、工程师
- 点击"详情"可查看完整追溯信息

**价值**：
- 责任可追溯
- 数据来源清晰
- 便于核实和审计

## 五、常见问题演示回答

### Q1：如果历史价格已过期怎么办？

A：系统默认查询近1年记录，减少过期数据的干扰。同时，查询结果会显示报价时间，用户可自行判断是否仍有参考价值。如价格确实需要更新，新的询价结果会补充到系统中，形成新的参考数据。

### Q2：如何确保上传数据的准确性？

A：
1. 系统要求填写部门和工程师信息，责任到人
2. 所有操作都有审计日志
3. 查询时可看到来源文件，便于核实
4. 建议配合内部管理制度，明确上传规范

### Q3：系统支持多少用户同时使用？

A：SQLite理论上支持多用户并发读，但写入时会锁定数据库。对于企业内部使用（几十人同时使用）完全足够。如需更高并发，可迁移到MySQL或PostgreSQL。

### Q4：数据安全如何保障？

A：
1. 所有数据存储在企业内部服务器
2. 数据库文件可定期备份
3. 所有操作都有审计日志
4. 建议配合企业网络权限管理
