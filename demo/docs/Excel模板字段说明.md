# 企业内部历史询价复用系统 - Excel模板字段说明文档

## 一、字段分类

### 1. 基础价格字段（必填）

这些字段构成每条询价记录的最小闭环，必须填写。

| 字段名 | 类型 | 是否必填 | 说明 | 示例值 |
|--------|------|----------|------|--------|
| 材料名称 | 文本 | **是** | 材料的通用名称 | 螺纹钢、水泥、砂石 |
| 规格型号 | 文本 | **是** | 材料的具体规格或型号 | HRB400E Φ16、P.O 42.5、中砂 |
| 单价 | 数字 | **是** | 询价单价（单位：元） | 4250、480、85 |
| 地区 | 文本 | **是** | 报价适用的地区 | 成都、绵阳、德阳 |
| 报价时间 | 日期 | **是** | 供应商报价的时间 | 2024-01-15、2024/1/15 |

### 2. 基础价格字段（推荐）

| 字段名 | 类型 | 是否必填 | 说明 | 示例值 |
|--------|------|----------|------|--------|
| 单位 | 文本 | 推荐 | 计量单位 | 吨、方、米、套、桶 |
| 是否含税 | 文本 | 推荐 | 价格是否含税 | 是、否 |
| 供应商/来源 | 文本 | 推荐 | 供应商名称或价格来源 | 四川攀钢集团、峨眉山水泥厂 |
| 备注 | 文本 | 可选 | 其他说明信息 | 项目急需、批量采购、现货供应 |

### 3. 责任字段（必填）

支持追责、统计与上传台账。

| 字段名 | 类型 | 是否必填 | 说明 | 示例值 |
|--------|------|----------|------|--------|
| 填报部门 | 文本 | **是** | 填写询价信息的部门 | 采购部、工程部、成本部 |
| 填报工程师 | 文本 | **是** | 填写询价信息的工程师姓名 | 张三、李四、王五 |
| 上传人 | 文本 | 可选 | 实际上传文件的人员，默认与填报工程师相同 | 张三、李四 |

### 4. 业务标识字段（推荐）

| 字段名 | 类型 | 是否必填 | 说明 | 示例值 |
|--------|------|----------|------|--------|
| 项目名称 | 文本 | 推荐 | 所属项目名称 | 成都天府新区道路改造项目 |
| 询价类别 | 文本 | 推荐 | 区分询价类型 | 项目询价、市场询价、供应商报价 |
| 序号 | 数字 | 可选 | 记录序号 | 1、2、3... |

## 二、字段填写规范

### 1. 日期格式

报价时间支持多种格式，系统会自动识别：
- 标准格式：2024-01-15
- 斜杠格式：2024/1/15
- 中文格式：2024年1月15日

**建议**：统一使用 `YYYY-MM-DD` 格式，如 `2024-01-15`

### 2. 价格格式

- 支持整数：4250
- 支持小数：4250.50
- 不需要填写单位（默认为"元"）
- 如果价格含税，在"是否含税"列填写"是"

### 3. 材料名称规范

建议统一材料名称，避免同一材料因名称不同而无法匹配：

| 不规范 | 规范 |
|--------|------|
| 螺纹钢、罗纹钢、钢筋 | 螺纹钢 |
| 水泥、硅酸盐水泥 | 水泥（在规格型号中详细说明） |
| 砂、沙子、河砂 | 砂石（在规格型号中区分中砂、细砂） |

### 4. 规格型号规范

规格型号应尽量详细和规范，便于匹配：

| 材料 | 规范的规格型号 |
|------|---------------|
| 螺纹钢 | HRB400E Φ16 |
| 水泥 | P.O 42.5 |
| 混凝土 | C30 |
| 沥青 | AH-70 |
| 管材 | DN300 钢管 |
| 电缆 | YJV-4×185 |

### 5. 地区规范

建议统一使用城市名称，不带"市"字：

| 不规范 | 规范 |
|--------|------|
| 成都市、成都地区 | 成都 |
| 绵阳市 | 绵阳 |
| 德阳市 | 德阳 |

## 四、常见问题

### Q1：列名不完全匹配怎么办？

系统支持多种常见列名变体：

| 标准列名 | 支持的变体 |
|----------|-----------|
| 材料名称 | 材料、名称 |
| 规格型号 | 规格、型号 |
| 单价 | 价格 |
| 是否含税 | 含税 |
| 供应商/来源 | 供应商、来源 |
| 地区 | 区域 |
| 报价时间 | 询价时间、时间 |
| 备注 | 说明 |
| 填报部门 | 部门 |
| 填报工程师 | 工程师、询价人 |

### Q2：有些列没有数据怎么办？

- **必填字段**必须填写，否则该行记录会被跳过
- **推荐字段**建议填写，缺失会影响后续查询和统计
- **可选字段**可以不填写或留空

### Q3：如何处理多个项目？

建议在"项目名称"列区分不同项目，这样可以：
- 在查询时按项目筛选
- 在统计时看到各项目的询价情况
- 便于后续按项目导出数据

### Q4：同一个材料有多个供应商报价怎么办？

可以分多行记录，每行一个供应商报价：

```
| 材料名称 | 规格型号 | 单价 | 供应商/来源    | 地区 |
|----------|----------|------|----------------|------|
| 螺纹钢   | HRB400E Φ16 | 4250 | 四川攀钢集团   | 成都 |
| 螺纹钢   | HRB400E Φ16 | 4300 | 成都钢铁厂     | 成都 |
| 螺纹钢   | HRB400E Φ16 | 4200 | 重庆钢铁厂     | 成都 |
```

这样可以在查询时看到不同供应商的价格对比。

### Q5：如何更新已有的询价记录？

上传新的Excel文件即可，系统会：
- 保留所有历史记录
- 新记录与历史记录独立存储
- 通过时间戳区分新旧数据

不建议删除或修改已上传的文件，保持历史记录的完整性。

## 六、数据质量要求

为确保系统有效运行，请遵守以下数据质量要求：

1. **准确性**：价格、规格等信息必须真实准确
2. **完整性**：必填字段不能为空
3. **规范性**：材料名称、规格型号应统一规范
4. **及时性**：询价完成后及时上传，确保数据的时效性
5. **可追溯性**：填报部门和工程师信息必须真实，便于追溯

## 七、数据安全说明

- 所有上传的数据存储在企业内部服务器
- 数据库定期备份
- 所有操作都有审计日志
- 仅授权人员可访问系统
- 数据仅用于企业内部询价管理，不得外传
