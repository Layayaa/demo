# 部署指南

## 一、概述

本文档详细说明企业内部历史询价复用系统的部署流程，供运维人员参考。

**系统要求**：
- 操作系统：CentOS 7.9+ 或 Ubuntu 20.04+
- Python：3.10+
- 数据库：MySQL 8.0 或 SQLite

## 二、服务器配置建议

### 2.1 推荐配置（~100人使用）

| 项目 | 配置 |
|------|------|
| CPU | 2核 或 4核 |
| 内存 | 4GB 或 8GB |
| 系统盘 | 40GB SSD |
| 数据盘 | 100GB SSD |
| 带宽 | 3-5Mbps |

### 2.2 安全组配置

| 端口 | 协议 | 说明 |
|------|------|------|
| 22 | TCP | SSH管理 |
| 80 | TCP | HTTP |
| 443 | TCP | HTTPS |
| 3306 | TCP | MySQL（内网） |

## 五、应用配置

### 5.1 创建目录结构

```bash
sudo mkdir -p /opt/inquiry-system/{app,data,logs,config,deploy}
sudo mkdir -p /opt/inquiry-system/data/{database,uploads}
sudo mkdir -p /opt/inquiry-system/logs/{nginx,gunicorn}

# 设置权限
sudo chown -R www-data:www-data /opt/inquiry-system
```

### 5.2 配置环境变量

```bash
# 创建环境变量文件
cat > /opt/inquiry-system/config/env << EOF
export SECRET_KEY="$(python3 -c 'import secrets; print(secrets.token_hex(32))')"
export DATABASE_URL="mysql+pymysql://inquiry:您的密码@localhost:3306/inquiry_system?charset=utf8mb4"
export FLASK_ENV="production"
EOF
```

### 5.3 修改应用配置

编辑 `/opt/inquiry-system/app/backend/config.py`（如需创建）:

```python
import os

class Config:
    SECRET_KEY = os.environ.get('SECRET_KEY', 'dev-secret-key')
    SQLALCHEMY_DATABASE_URI = os.environ.get('DATABASE_URL', 'sqlite:///database/inquiry_file.db')
    SQLALCHEMY_TRACK_MODIFICATIONS = False
    MAX_CONTENT_LENGTH = 16 * 1024 * 1024
    SESSION_COOKIE_HTTPONLY = True
    SESSION_COOKIE_SECURE = True
    PERMANENT_SESSION_LIFETIME = 86400  # 24小时
```

## 六、Nginx配置

### 6.1 创建配置文件

```bash
sudo cp /opt/inquiry-system/deploy/config/nginx.conf.example /etc/nginx/sites-available/inquiry-system
sudo ln -s /etc/nginx/sites-available/inquiry-system /etc/nginx/sites-enabled/
```

### 6.2 配置内容

```nginx
# /etc/nginx/sites-available/inquiry-system

upstream inquiry_app {
    server 127.0.0.1:5000 fail_timeout=0;
}

server {
    listen 80;
    server_name price.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name price.example.com;

    # SSL证书（使用Let's Encrypt）
    ssl_certificate /etc/letsencrypt/live/price.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/price.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 上传文件大小限制
    client_max_body_size 20M;

    # 静态文件
    location /static/ {
        alias /opt/inquiry-system/app/frontend/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 代理到Flask应用
    location / {
        proxy_pass http://inquiry_app;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 300s;
        proxy_read_timeout 300s;
    }

    # 日志
    access_log /opt/inquiry-system/logs/nginx/access.log;
    error_log /opt/inquiry-system/logs/nginx/error.log;
}
```

### 6.3 测试并重启

```bash
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx
```

## 七、Gunicorn配置

### 7.1 创建配置文件

```bash
cp /opt/inquiry-system/deploy/config/gunicorn.conf.py.example /opt/inquiry-system/config/gunicorn.conf.py
```

### 7.2 配置内容

```python
# /opt/inquiry-system/config/gunicorn.conf.py

import multiprocessing

bind = "127.0.0.1:5000"
workers = multiprocessing.cpu_count() * 2 + 1  # 推荐配置
worker_class = "gevent"
timeout = 120
keepalive = 5
max_requests = 1000
max_requests_jitter = 100

accesslog = "/opt/inquiry-system/logs/gunicorn/access.log"
errorlog = "/opt/inquiry-system/logs/gunicorn/error.log"
loglevel = "info"

# 进程名
proc_name = "inquiry-system"
```

## 八、Supervisor配置（进程管理）

### 8.1 安装Supervisor

```bash
# Ubuntu
sudo apt install supervisor

# CentOS
sudo yum install supervisor
```

### 8.2 创建配置文件

```bash
sudo cp /opt/inquiry-system/deploy/config/supervisor.conf.example /etc/supervisor/conf.d/inquiry-system.conf
```

### 8.3 配置内容

```ini
# /etc/supervisor/conf.d/inquiry-system.conf

[program:inquiry-system]
directory=/opt/inquiry-system/app/backend
command=/opt/inquiry-system/venv/bin/gunicorn -c /opt/inquiry-system/config/gunicorn.conf.py app:app
user=www-data
autostart=true
autorestart=true
startsecs=10
stopwaitsecs=600
stdout_logfile=/opt/inquiry-system/logs/app.out.log
stderr_logfile=/opt/inquiry-system/logs/app.err.log
environment=PATH="/opt/inquiry-system/venv/bin"
```

### 8.4 启动服务

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start inquiry-system
```

## 十二、故障排查

### 12.1 502 Bad Gateway

1. 检查Gunicorn是否运行：`sudo supervisorctl status`
2. 检查端口占用：`netstat -tlnp | grep 5000`
3. 检查日志：`tail -f /opt/inquiry-system/logs/app.err.log`

### 12.2 数据库连接失败

1. 检查MySQL服务：`sudo systemctl status mysql`
2. 检查用户权限：`mysql -u inquiry -p`
3. 检查环境变量：`echo $DATABASE_URL`

### 12.3 文件上传失败

1. 检查目录权限：`ls -la /opt/inquiry-system/data/uploads/`
2. 检查Nginx配置：`client_max_body_size`
3. 检查磁盘空间：`df -h`

## 十三、安全加固

### 13.1 系统安全

```bash
# 禁用root SSH登录
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config

# 配置防火墙
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
```

### 13.2 应用安全

- 确保 `SECRET_KEY` 使用强随机值
- 定期更新依赖：`pip install --upgrade -r requirements.txt`
- 启用HTTPS强制跳转
- 配置适当的文件权限
