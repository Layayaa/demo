# 询价系统启动指南（更新于 2026-04-13）

## 1. 适用范围

本指南适用于本地开发环境（Windows），项目目录：

- `C:\Users\LAYA\demo-main`

当前后端默认监听端口：

- `5001`

访问地址：

- `http://127.0.0.1:5001`

---

## 2. 一键启动（推荐）

在项目根目录双击运行：

- `start_new.bat`

脚本会自动完成：

1. 检查 Python（`py` 启动器）
2. 检查/创建虚拟环境 `.venv`
3. 安装依赖（缺失时）
4. 启动后端服务（`backend/app.py`）

启动成功后，控制台会显示类似：

- `访问地址: http://localhost:5001`

---

## 3. 手动启动（命令行）

在 PowerShell 中执行：

```powershell
cd C:\Users\LAYA\demo-main

# 1) 创建虚拟环境（首次）
py -3.12 -m venv .venv

# 2) 安装依赖
.\.venv\Scripts\python.exe -m pip install -r requirements.txt

# 3) 启动服务
cd backend
..\.venv\Scripts\python.exe app.py
```

浏览器访问：

- `http://127.0.0.1:5001`

---

## 5. 数据库与文件位置

项目使用 SQLite 四库分离：

- `database/inquiry_file.db`（文件信息）
- `database/price_record.db`（询价明细）
- `database/upload_audit.db`（上传审计）
- `database/user.db`（用户与绑定）

上传文件目录：

- `uploads/`

## 6. 常见问题排查

### Q1：浏览器显示“无法访问网站”

先确认后端是否在运行：

- 控制台是否有 `访问地址: http://localhost:5001`
- 使用 `http://127.0.0.1:5001` 访问（不要用旧端口 5000）

### Q2：提示依赖缺失（ModuleNotFoundError）

执行：

```powershell
cd C:\Users\LAYA\demo-main
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### Q3：5001 端口被占用

可临时换端口启动：

```powershell
cd C:\Users\LAYA\demo-main\backend
$env:PORT='5002'
..\.venv\Scripts\python.exe app.py
```

然后访问：`http://127.0.0.1:5002`

### Q4：登录失败（确认账号密码无误仍失败）

排查顺序：

1. 确认连接的是当前项目实例（`demo-main`）
2. 检查 `database/user.db` 是否被误替换
3. 重启服务后重试

### Q5：上传成功但数据不显示

排查：

1. 在“已上传文件列表”确认文件 `parse_status` 是否为 `success`
2. 在查询页确认时间范围（默认近 1 年）
3. 检查 `database/price_record.db` 是否有记录

---

## 7. 重置本地环境（谨慎）

仅用于本地演示重置，不可用于生产：

1. 停止后端服务
2. 备份 `database/` 与 `uploads/`
3. 按需清理后重启服务

重启后系统会重新初始化基础结构；若用户表为空会重建默认管理员。

---

## 8. 当前版本启动检查清单

每次启动后建议确认：

1. 页面可访问：`http://127.0.0.1:5001`
2. 可正常登录管理员
3. 可上传 Excel 并解析成功
4. 查询与详情可正常打开
5. 统计分析接口可返回数据
