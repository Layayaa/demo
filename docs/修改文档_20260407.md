# 企业内部历史询价复用系统 - 修改文档

**文档日期**: 2026-04-07

---

## 一、系统概述

企业内部历史询价复用系统，用于统一管理历史询价文件，避免重复询价，沉淀历史成果，提升团队效率。

### 技术架构
- **后端**: Flask + SQLAlchemy
- **前端**: Bootstrap 5 + 原生JavaScript
- **数据库**: SQLite (三库分离架构)

### 2.6 填报工程师智能识别（新增）

#### 修改位置
- 文件: `backend/template_config.py` (第437-509行)

#### 功能说明
当Excel文件中没有明确的"填报工程师"列时，系统会自动智能识别工程师列。

#### 识别策略

| 策略 | 说明 |
|------|------|
| 列名映射 | 匹配"工程师"、"询价人"、"经办人"等关键词 |

| Excel列名 | 识别方式 |

### 2.7 入库前去重功能（新增）

#### 修改位置
- 文件: `backend/app.py` (`is_record_exists` 函数)

#### 功能说明
上传Excel时，系统会在入库前检查每条记录是否已存在，自动跳过重复记录。

#### 去重规则

判断记录是否重复的条件（必须全部满足）：

| 字段 | 匹配规则 |
|------|---------|
| 材料名称 | 完全相同 |
| 规格型号 | 完全相同（或都为空） |
| 供应商 | 完全相同（或都为空） |
| 报价时间 | 完全相同（或都为空） |

#### 效果展示

上传重复文件时显示：
```
上传成功，共解析50条记录，跳过12条重复记录
```

智能识别报告：
- 成功解析：50 条记录
- 重复跳过：12 条（材料+规格+供应商+日期相同）

#### 统计去重

统计页面中的数据也进行了去重处理：

| 统计项 | 去重规则 |
|--------|----------|
| 总记录数 | 按"材料+规格+供应商+报价日期"组合去重 |
| 按工程师统计 | 按"工程师+材料+规格+供应商+报价日期"组合去重 |
| 按材料统计 | 按"材料+规格+供应商+报价日期"组合去重 |

**说明**：同一工程师多次上传相同材料只算一条，不同工程师上传相同材料分别计数。

### 2.9 自然语言解析模块重构（新增）

#### 修改位置
- 文件: `backend/nlp_parser.py` (新建)

#### 功能说明
重构自然语言查询解析模块，采用专业化架构设计。

#### 架构设计

```
用户输入 → 预处理 → 分词 → 意图识别 → 实体识别 → 参数校验 → 返回结构化参数
```

#### 核心功能

| 模块 | 说明 |
|------|------|
| 预处理 | 全角转半角、去除查询前缀 |
| 分词 | 支持 jieba 分词，降级时使用正则分词 |
| 意图识别 | price_inquiry（询价）、comparison（比价）、trend（趋势）、statistics（统计）|
| 实体提取 | 材料名称、规格型号、地区、时间范围 |

#### 支持的时间表达

| 表达方式 | 示例 |
|---------|------|
| 相对日期 | 今天、昨天、本周、上月、今年 |
| 最近N天/月 | 最近7天、最近三个月、近半年 |
| 日期范围 | 3月到5月、2024年到2025年 |
| 具体日期 | 2024-01-15、3月20日 |

#### 测试示例

| 输入 | 材料 | 规格 | 地区 | 时间 | 意图 |
|------|------|------|------|------|------|
| 成都最近一个月钢筋C30的价格 | 钢筋 | C30 | 成都 | 最近1个月 | price_inquiry |
| 门的价格 | 门 | - | - | 近1年（默认） | price_inquiry |
| 上个月钢材趋势 | 钢材 | - | - | 上个月 | trend |
| 成都C30混凝土对比 | 混凝土 | C30 | 成都 | - | comparison |

### 2.10 用户登录模块（新增）

#### 修改位置
- 文件: `backend/models.py` - 新增 User 模型
- 文件: `backend/app.py` - 添加认证逻辑和 API
- 文件: `frontend/login.html` - 新建登录页面
- 文件: `frontend/index.html` - 添加用户区域和管理功能

#### 功能说明
为系统添加用户认证和权限管理功能。

#### 技术方案

| 项目 | 选择 | 说明 |
|------|------|------|
| 认证方式 | Flask-Login + Session | 轻量级，适合单机部署 |
| 密码加密 | bcrypt | 安全性高，防彩虹表攻击 |
| 数据库 | SQLite（user.db） | 与现有三库分离架构保持一致 |

#### 数据库设计

新增用户数据库 `user.db`，包含 User 表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键，自增 |
| phone | String(11) | 手机号，唯一 |
| password_hash | String(128) | bcrypt 加密密码 |
| real_name | String(100) | 真实姓名 |
| department | String(100) | 部门 |
| role | String(20) | 角色：admin/user |
| is_active | Boolean | 账号状态 |
| created_at | DateTime | 创建时间 |
| last_login | DateTime | 最后登录时间 |

#### API 接口

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| /api/login | POST | 无 | 登录 |
| /api/logout | POST | 已登录 | 退出 |
| /api/user/info | GET | 已登录 | 获取当前用户信息 |
| /api/user/password | POST | 已登录 | 修改密码 |
| /api/admin/users | GET | admin | 用户列表 |
| /api/admin/users | POST | admin | 添加用户 |
| /api/admin/users/:id | PUT | admin | 编辑用户 |
| /api/admin/users/:id | DELETE | admin | 删除用户 |
| /api/admin/users/:id/reset-password | POST | admin | 重置密码 |

#### 角色权限

| 角色 | 权限 |
|------|------|
| admin | 添加/编辑/删除用户、重置密码、查询价格 |
| user | 查询价格、修改自己的密码 |

#### 默认管理员账号

系统首次启动时自动创建默认管理员：

| 字段 | 值 |
|------|-----|
| 手机号 | 13800138000 |
| 密码 | 138000（手机号后六位）|
| 角色 | admin |
| 姓名 | 系统管理员 |

#### 安全措施

1. **密码安全**：bcrypt 加密，不存明文
2. **Cookie 安全**：HttpOnly 标志，防止 XSS 攻击
3. **登录安全**：统一错误提示"手机号或密码错误"，不暴露具体原因
4. **权限安全**：所有接口验证登录状态，管理员接口验证角色
5. **API 权限分离**：
   - 页面路由使用 `@login_required`，未登录重定向到登录页
   - API 接口使用 `@api_login_required`，未登录返回 401 JSON 响应

#### 前端界面

1. **登录页** - 手机号/密码输入，支持回车提交
2. **导航栏用户区** - 显示用户名、头像首字
3. **下拉菜单** - 修改密码、退出登录
4. **用户管理弹窗** - 管理员可见，支持添加/编辑/删除用户、重置密码
